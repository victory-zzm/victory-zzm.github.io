javascript中的浅拷贝和深拷贝

发表于2021-09-06|更新于2021-09-07|博客笔记
|阅读量:

1.浅拷贝和深拷贝的区别

  • 浅拷贝：进行浅拷贝时，如果数据是基本数据类型，那么就如同直接赋值，会拷贝其本身，但是如果拷贝的内容出类基本数据类型还有另外一层对象，则浅拷贝只是拷贝了引用（即是拷贝了地址），对象的改变也会导致拷贝对象的改变；
  • 深拷贝：深拷贝会拷贝多层，即使是嵌套了对象也会拷贝出来；

2.实现浅拷贝

Object.assign();

  • Object.assign() 方法可以把任意多个的源对象自身的可枚举属性拷贝给目标对象，然后返回目标对象。但是 Object.assign() 进行的是浅拷贝，拷贝的是对象的属性的引用，而不是对象本身;

  • ```js
    Object.assign(target, …sources);

  • 浅拷贝就是拷贝了一层，除了对象是拷贝的引用类型，其他都是直接将值传递，有自己的内存空间的

方法二，代码如下：

  • ```js
    var obj1 = {

    a: "hello",
b: {
    a: "hello",
    b: 21}

    };

    var cloneObj1= Object.assign({}, obj1);
    cloneObj1.a = “changed”;
    cloneObj1.b.a = “changed”;
    console.log(obj1.a); //hello(没有影响到原对象)
    console.log(obj.b.a); // “changed”(影响到原对象，可见对于第二层是浅拷贝)

    • 坏处：
    1. 它会抛弃对象的constructor。也就是深拷贝之后，不管这个对象原来的构造函数是什么，在深拷贝之后都会变成Object。
    2. Date对象, RegExp对象, Error对象等是无法通过这种方式深拷贝。这种方法能正确处理的对象只有 Number, String, Boolean, Array, 扁平对象，即那些能够被 json 直接表示的数据结构。
    3. 如果原对象中有值为undefined的情况, JSON.stringify 后会丢失
    4. 如果obj里有NaN、Infinity和-Infinity，则序列化的结果会变成null

    也就是说，只有可以转成JSON格式的对象才可以这样用，像function没办法转成JSON。
